# 노냐? 홍보 스레드 원고

## 한 줄 소개

밤새 Claude/Codex가 질문 하나에 멈춰 있던 사람을 위한 AI 세션 자동복구 도구. 멈추면 깨우고, 애매하면 위험하게 승인하지 않는다.

## 게시용 스레드

### 1

AI한테 "밤새 해놔" 하고 잤는데, 아침에 보면 한 줄 질문에서 멈춰 있던 적 있나요?

`노냐?`는 그 순간을 잡으려고 만든 오픈소스 AI 세션 watchdog입니다.

### 2

핵심은 단순합니다.

Claude, Codex, Antigravity 세션이 에러, rate limit, idle, 입력대기, 미검증 완료 상태로 멈췄는지 보고, 안전하면 같은 창 또는 같은 tmux pane에 이어가기 신호를 보냅니다.

### 3

새 headless job을 만들지 않습니다.

이미 쓰던 대화, 구독 표면, 컨텍스트를 그대로 살립니다. 밤새 작업의 맥락을 버리지 않고, 멈춘 자리에서 다시 밀어주는 방식입니다.

### 4

v0.2.1에서 제일 큰 변화:

자율 모드가 입력대기에서 놀지 않습니다.

로컬 지침에 답이 있으면 답하고, 없으면 "가장 안전하고 되돌릴 수 있는 로컬 기본값으로 계속"이라고 답해 루틴 질문을 넘깁니다.

### 5

그렇다고 아무거나 승인하지 않습니다.

secrets, billing, 삭제/초기화, 패키지 설치, 외부 네트워크, 권한 상승, production/deploy/publish/release는 자동 승인하지 않습니다. 안전한 로컬 검증이나 dry-run으로 돌립니다.

### 6

가장 믿을 만한 경로는 CLI + tmux입니다.

Claude CLI / Codex CLI는 포커스가 없어도 정확한 pane에 전달합니다. GUI 앱은 실계정 보호를 위해 기본 alert-only이고, 실제 입력은 명시 opt-in입니다.

### 7

macOS 앱도 있습니다.

메뉴바에 작은 "노냐?"가 떠서 세션 상태를 보여주고, 멈춤/복구/검증 상태를 로컬 JSON 상태 파일로 따라갑니다. 코어는 Python stdlib 기반이라 런타임 의존성이 없습니다.

### 8

받아보기:

GitHub: https://github.com/ezBuilder/nonya

Release: https://github.com/ezBuilder/nonya/releases/tag/v0.2.1

"AI가 밤새 일하다가 중간에 자는 문제"가 짜증났던 사람이라면, 이건 꽤 바로 체감됩니다.

## 짧은 소개글

`노냐?`는 Claude/Codex/Antigravity 작업 세션이 멈췄는지 감시하고, 안전하면 같은 창이나 tmux pane에 이어가기 신호를 보내는 오픈소스 자동복구 도구입니다. v0.2.1부터 자율 모드 입력대기도 로컬 지침 또는 안전한 기본 응답으로 처리해, 사용자가 자는 동안 질문 하나 때문에 세션이 멈춰 있는 상황을 줄입니다.

## 카드뉴스 문구

3. 새 작업을 만들지 않고, 쓰던 대화를 그대로 살립니다.
4. v0.2.1: 자율 모드 입력대기도 알아서 처리합니다.

6. CLI + tmux는 정확한 pane으로 직접 전달합니다.
7. macOS 메뉴바 앱으로 상태를 바로 봅니다.
