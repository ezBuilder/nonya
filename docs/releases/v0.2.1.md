# nonya v0.2.1

자율 모드가 이제 입력대기에서 멈춰 서지 않습니다.

## Highlights

- `--mode auto`에서 평문 질문과 `AskUserQuestion`류 입력대기를 처리합니다.
- 로컬 지침(`AGENTS.md`, `CLAUDE.md`, `README.md`)에 명확한 답이 있으면 그 답을 사용합니다.
- 답이 없으면 "가장 안전하고 되돌릴 수 있는 로컬 기본값으로 계속"이라는 보수적 응답을 주입합니다.
- secrets, billing, destructive, install, external network, privilege, production, deploy, publish, release 요청은 승인하지 않고 안전 경로로 우회시킵니다.
- 카드뉴스 PNG를 함께 제공합니다.

## Assets

- `nonya-0.2.1.dmg`: macOS 앱 배포 파일. Developer ID 서명은 적용되어 있으며, 이 빌드 머신에는 `notarytool` keychain profile이 없어 Apple notarization은 pending입니다.
- `nonya-0.2.1-windows-x64.zip`: Windows CLI 패키지
- `nonya-cardnews-v0.2.1.zip`: 홍보용 카드뉴스 PNG

## Install

1. DMG를 다운로드합니다.
2. `Nonya.app`을 Applications로 드래그합니다.
3. 첫 실행 후 macOS Accessibility 권한을 허용합니다.
4. Claude/Codex CLI는 tmux 경로가 가장 안정적입니다.

## Safety

GUI 앱 개입은 보수적으로 게이트됩니다. Watch all 스캐너는 사용자가 자리를 비웠고 OCR로 대상 대화를 증명할 수 있을 때만 조건부로 개입하며, 대상이 애매하면 알림만 보냅니다. 단일 세션 직접 앱 주입과 실계정 스모크 테스트에는 `NONYA_ALLOW_REAL_APP_INJECT=1`이 필요합니다.

## Maintainer Note

완전한 Gatekeeper 무경고 배포를 위해서는 `xcrun notarytool store-credentials`로 notarization profile을 저장한 뒤 `packaging/sign-notarize.sh`를 다시 실행해야 합니다.
